## 시도 (1회)

- **필터1: 실패 ❌ (V1 한계 노출).** 제목 "부린 왕자"는 '부동산+어린왕자'의 **조어**라 토큰이 DB 어느 제목과도 2개 이상 안 겹침 → 매칭 0건. 후보가 082(650.1) 하나뿐이고, 사서가 검토한 179.9·332.6324·811.36은 **아예 안 올라옴.** 정답이 후보에 없으니 2단계가 아무리 잘해도 구조적으로 못 맞힘.
- **필터2: 빈약한 입력에도 안전하게 행동 ✅.** 650.1을 저확신(0.45)으로 두고, `needs_toc`(메타 부재로 초점 불가) + `cross_major`(소설 800 / 부동산 330으로 갈릴 수 있음)로 **에스컬레이션**(ambiguity 0.80). "모른다"고 사람에게 넘김 = 상 케이스에 맞는 동작.

## 결론

**병목은 필터1(retrieve recall).** 조어 제목 + 메타데이터 부재 → 후보 생성 실패. 필터2는 그 위에서 최선(저확신+에스컬레이션)을 함.
- **평가 관점:** 상 케이스는 Exact로 재면 필연 실패(정답이 후보에 없거나 사람도 갈림). 유의미한 성공 지표는 **"에스컬레이션을 제대로 띄웠나"** → ✓.
- **개선 방향:** ① 입력에 부제/책소개/키워드 보강 시 필터1이 후보를 건지나, ② V2(임베딩)가 조어 제목의 의미('부동산·성찰')를 잡나 — 두 실험의 직접 대상.
