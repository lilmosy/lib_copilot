# AGENTS.md — lib_copilot 작업 규약

AI 에이전트(및 사람)가 이 저장소에서 작업할 때 지키는 규칙. 설계 배경은 `docs/design.md`.

## 프로젝트 한 줄 요약
실제 사서의 청구기호(852 ▼h) 부여 업무를 파이프라인으로 옮기고, 판단이 갈리는 지점만 사람에게 넘기는(HITL) 협업 에이전트. 시나리오 골든셋으로 평가하며 고도화한다.

## 아키텍처 불변식 (깨지 말 것)
- **데이터 계약은 `src/schema.py` 한 곳.** 단계 간에는 스키마 타입만 주고받는다. 내부 구현을 바꿔도 배선(`pipeline.py`)은 불변.
- **funnel 2단계** (설계: design.md §2): ① 1차 `retrieve_*`(recall, 넓게) → `RetrieveResult` → ② 2차 `classify`(precision, LLM).
- **1차 검색기는 두 구현, 같은 계약.** `retrieve_token`(토큰)·`retrieve_embed`(임베딩) 모두 `retrieve(book) -> RetrieveResult`. **classify/pipeline은 공유(복제 금지)**. `config.RETRIEVER`로 스위치.
  - *복제 금지가 무슨 뜻인가:* V2를 만들 때 `pipeline_embed.py`·`classify_embed.py`를 새로 만들지 말 것. 파일이 2벌 되면 한쪽만 고쳐 조용히 어긋난다. **다른 부분(`retrieve_embed.py`)만 새로 만들고 나머지는 하나를 둘이 공유한다.** 이게 가능한 이유가 "같은 계약"이다.
- **LLM 호출은 `classify.py` 한 곳에만.** 나머지는 순수 파이썬. 1차를 LLM으로 만들지 말 것(비용·scale) — 1차는 토큰/임베딩.
- **토큰 규칙은 `src/tokens.py` 한 곳.** `retrieve_token`(평가 경로)과 `db`(데모 경로)가 공유한다. `MIN_OVERLAP`의 주인은 `config.py`. 어느 한쪽에 복사하지 말 것 — 두 경로가 어긋나면 "데모에서 본 것"과 "평가 점수"가 다른 것을 가리킨다.
- **저장소 접근은 `src/db.py` 한 곳. DataFrame은 그 파일 밖으로 내보내지 않는다.** 밖으로는 `CollectionBook`/`DistBucket`만. (pandas → SQLite 교체 시 부르는 쪽 불변. design.md §4.1)
- **주제 우선 · 개수 보조.** classify 프롬프트는 shelf_sample로 주제를 판단하게 하고 개수는 보조로만. 이유: 개수는 상한 샘플이라 부정확 + 개수최다≠정답.
- **082는 선택 입력(prior).** 있으면 후보에 포함해 유지/기각을 LLM이 판단. 없다고 실패하지 않는다.
  - 데모 경로에서는 **신분이 셋**이다: 미판정(맨 앞) / 정합함(채택·종료) / **기각(후보엔 남되 1순위 자리 뺌)**. 기각된 082를 후보에서 **지우지 말 것** — 근거가 사라지고 LLM이 반박할 여지도 없어진다. (design.md §5.1)
- **단일 정답을 강제하지 않는다.** 에스컬레이션 신호가 있으면 후보+근거를 제시하고 `escalate=true`.
- **평가 경로는 동결.** `retrieve_token`·`classify`의 프롬프트를 바꾸면 골든셋 점수가 왜 변했는지 알 수 없게 된다. 데모용 기능은 **기본값이 꺼진 선택 인자**로 넣을 것(예: `classify(..., human_notes=None)`). 손댔으면 `output/*_retrieve.json`과 대조해 회귀가 없음을 확인한다.
- **중간산출물은 `output/`에 json 덤프**(사람이 검수). 단계 간 전달은 메모리 객체(파일 불필요).

## 코드 스타일
- Python 3.12, 타입힌트 사용, `from __future__ import annotations`.
- 검증 모델은 pydantic v2 (`BaseModel`).
- 주석/문서는 한국어. 기존 파일의 톤(간결한 설명 + 근거)을 따른다.
- 새 의존성은 `requirements.txt`에 추가하고 이유를 devlog에.

## 데이터 규약
- **정답셋:** `data/scenarios/<caseNN_slug>.json` — `input`(BookInput) / `expected`(정답) / `human_trace`(사서 실제 검색) 3블록.
  - `expected`는 `writer_final`(작성자 확정)과 `review_final`(교열 최종)을 **분리**해 담는다. 상 케이스는 둘이 다르다.
  - ⚠️ `input.keywords`는 **수기로 넣은 것**이다. 실전에는 없으며 이게 데모의 최대 난제다(design.md §7.1).
- **DB:** `data/collection.json` — 본교 장서 1682건(청구기호 크롤). 기존 10필드 + `detail`(상세페이지 원자료).
  - `detail`의 **필드를 미리 고르지 않는다.** 라벨이 23종 이상이고 책마다 다르며(3~98%) 재크롤이 3시간이다. th/td를 통째로 담고 해석은 나중에.
  - 저장은 전부, **프롬프트 투입은 선별**. 후보 3종 × 40권이라 상세 필드를 다 넣으면 2~3만 토큰이 되고, `형태사항`·`ISBN`·`언어`는 주제 판단과 무관해 노이즈만 된다. 한 필드씩 붙여 골든셋으로 효과를 측정할 것.
- **결정 로그:** `runs/decisions_YYYYMMDD.jsonl` — 데모에서 사서가 내린 최종 결정.
  - ⚠️ **결정 결과를 `collection.json`에 되쓰지 말 것.** 정답이 코퍼스에 섞여 데이터 누출을 매번 새로 만든다(design.md §8). 앱은 읽기 전용이다.
- `data/`는 커밋 대상이다(collection.json 600KB→수 MB). 단 `.env`·로그·캐시는 .gitignore.

## 로깅 (두 층으로 분리)
- **`docs/devlog.md` (사람용):** 왜 바꿨나 — 프롬프트 수정, 유사도 기준 변경, 실패한 시도와 이유. 시간순 append.
- **`runs/<타임스탬프>.jsonl` (기계용):** 케이스별 예측/정답/신호. 정량 개선 추적용.
- ⚠️ 스크립트 내부에서 `Date.now()`류 시간 함수에 의존하지 말 것 — 타임스탬프는 호출부(shell)에서 주입.

## 평가 우선
- 코드를 바꾸면 `python eval/evaluate.py`로 골든셋 지표(Exact/Prefix/Top-k)를 확인한다.
- baseline("검색 분포 최빈 ▼h") 대비 LLM+규칙이 얼마나 기여하는지 항상 비교한다.

## 설계 질문 — 결정됨 (상세: design.md §7)
- 제목 유사도: funnel로 우회(1차 recall 넓게, 2차 LLM 판단). 방법은 토큰(V1)/임베딩(V2) 두 버전 → 평가로 결정.
- 주제 판단: 후보 번호대의 shelf_sample로. **개수 아니라 주제 우선.**
- 저장소: **JSON 원본 + pandas 런타임.** SQLite는 아직 아님(design.md §4.1에 갈아탈 기준 명시).
- 데모 UI: Streamlit. 탭이 아니라 `st.radio`로 단계를 고른다 — `st.tabs`는 서버가 프로그램으로 전환시킬 수 없어 "정합 안 함 → 3단계로"가 동작하지 않는다.
- 남은 것: **키워드 출처(§7.1, 최우선)**, 임베딩 백엔드(V2), shelf_sample 대표성, 상 케이스 골든셋 확충.

## 크롤링 규약
- 학교 서버다. 요청 간 `DELAY_SEC` 유지, `verified` 쿠키는 5분마다 재발급.
- **오래 걸리는 크롤은 반드시 중간 저장한다.** 상세페이지는 건당 ~7초라 전체 3시간이고, 끝에만 저장하면 도중에 죽을 때 전부 날아간다. `--resume`은 중간 저장이 있어야 의미가 있다.
- 전체 크롤(1682건)을 돌리기 전에 `--limit`로 소량 확인할 것.

## 하지 말 것
- 시나리오 전제를 벗어난 임의 데이터로 정답을 만들지 않는다(정답은 케이스 분석 문서 기준).
- ▼i(도서기호)·▼m(부가기호)·852 조립은 Ver 1 스코프 밖. 지금 손대지 않는다.
- KDC를 DDC로 변환하지 않는다. 두 체계는 1:1 대응이 아니며 변환은 근거 없는 번호를 지어내는 것이다.
